منصة آلين v3.0.0 — نسخة الإنتاج الموحدة
========================================

هذه النسخة موحدة وليست مجموعة ترقيعات.
داخلها ملف قاعدة بيانات واحد فقط:
ALIN_V3_PRODUCTION_MASTER.sql

مهم قبل البدء
-------------
- هذه الترقية مخصصة لقاعدة منصة آلين الحالية بعد إصدار v2.8.0.
- خذ نسخة احتياطية من Supabase قبل التنفيذ.
- لا تعيد تشغيل ملفات SQL القديمة ولا تدمجها مع هذا الإصدار.

طريقة التطبيق
-------------
1) افتح Supabase ثم SQL Editor.
2) نفّذ ملف ALIN_V3_PRODUCTION_MASTER.sql كاملاً مرة واحدة.
3) يجب أن تظهر في آخر النتيجة أعمدة الجاهزية بقيمة true، والعدادان:
   invalid_audit_rows = 0
   expired_student_sessions = 0
4) انشر وظائف Supabase الموجودة داخل مجلد supabase/functions:
   - secure-login
   - admin-create-account
   - admin-update-account
   - admin-reset-password
   - admin-delete-account
5) أصل الموقع الحالي على GitHub Pages مضاف مسبقاً إلى CORS.
   عند استخدام دومين جديد أضفه في سر Supabase باسم ALIN_ALLOWED_ORIGINS.
   يمكن وضع أكثر من دومين مفصولاً بفاصلة.
6) ارفع ملفات هذه النسخة إلى GitHub بدلاً من ملفات النسخة القديمة.
7) بعد انتهاء النشر اضغط Ctrl + F5، ثم سجل خروجاً وادخل من جديد.

ما تغير في الإصدار
------------------
- سجل تدقيق محفوظ في قاعدة البيانات ولا يستطيع المتصفح تزويره أو حذفه.
- تسجيل الدخول يمر عبر Edge Function محمية بمحاولات محددة وربط الجهاز وعنوان الاتصال.
- كلمات مرور حسابات العمل لا تقل عن 12 حرفاً وتتضمن حروفاً وأرقاماً.
- مدير أعلى وصلاحيات منفصلة لبقية المديرين.
- أرشفة الحساب بدلاً من حذفه؛ تبقى طلباته وتسوياته القديمة محفوظة.
- حساب الطالب وطلباته انتقلت من التخزين المحلي إلى خدمات آمنة في Supabase.
- المدرس يرى بيانات البيع اللازمة فقط ولا تصله أرقام الطلاب أو عناوينهم.
- بيانات الطلبات والحسابات والمالية لا تُحفظ في Local Storage.
- النسخة الاحتياطية تشمل الكتالوج والإعدادات فقط، بدون بيانات شخصية أو مالية.
- قراءة الإشعارات أصبحت منفصلة لكل حساب.
- تثبيت إصدار Supabase المستخدم، وتضييق CORS ورسائل الأخطاء.
- تقليل تحميل JavaScript إلى ملفين محليين لكل واجهة بدلاً من عشرات الملفات.

اختبار سريع بعد النشر
---------------------
- جرّب كلمة مرور خاطئة عدة مرات وتأكد من إيقاف المحاولات مؤقتاً.
- جرّب حساب طالب: إنشاء، دخول، تعديل، عرض الطلبات، خروج.
- جرّب حساب مدرس وتأكد من عدم ظهور هاتف الطالب أو عنوانه.
- سلّم طلباً واحداً وتأكد أن القيد المالي لا يتكرر عند الضغط مرتين.
- أرشف حساباً تجريبياً وتأكد أنه لا يدخل، مع بقاء طلباته القديمة.
- اختبر الكمبيوتر والموبايل بعد تحديث قوي.

الفحص المنفذ
------------
- فحص تركيب 75 ملف JavaScript.
- فحص 66 وحدة وربط ترتيبها.
- 99 فحصاً للحماية والخصوصية والإصدار النهائي.
- فحص TypeScript لجميع وظائف Edge.
- اختبار تحميل محلي: 300 طلب متزامن على الملفات الثابتة، بتزامن 50، بدون أخطاء.

ملاحظة صريحة
------------

واختبار الضغط على قاعدة البيانات الحية يجب تأكيدهما بعد النشر؛ الاختبار المحلي لا يمثل
سعة خطة Supabase أو سرعة الإنترنت الفعلية.
