# ALIN v2.1.8 — إصلاح تعارض تريغر حماية الطلبات مع ترقية قاعدة البيانات

- سبب الخطأ: تريغر `alin_orders_protect_update` كان يعمل أثناء تحديث بيانات الطلبات القديمة داخل ملف الترقية، بينما جلسة SQL Editor ليست جلسة مستخدم تطبيق مربوط بحساب مدير.
- التحديث الحقيقي: ملف الترقية يوقف التريغر داخل نفس معاملة PostgreSQL قبل تحديث بيانات الطلبات، ثم يعيد إنشاءه بعد اكتمال الترقية.
- حماية الاستخدام العادي لا تُلغى؛ فحص الجاهزية يتأكد من رجوع التريغر بعد التحديث.
- عند فشل أي خطوة، المعاملة تتراجع تلقائيًا ولا تترك جدول الطلبات بلا حماية.
