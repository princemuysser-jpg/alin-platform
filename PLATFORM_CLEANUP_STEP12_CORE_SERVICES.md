# ALIN v2.2.7 — PLATFORM STEP 12

## التنفيذ الفعلي

تم تحويل `modules/core/platform.js` إلى نواة تشغيل صغيرة فقط، مع حذف التنفيذ القديم للإعدادات والهوية البصرية والتخزين والنسخ الاحتياطي وحساب الطالب والوظائف المشتركة.

### المالك الجديد لكل خدمة

- `modules/core/ui.js`: التهريب الآمن للنصوص، تنسيق الأموال، إنشاء المعرّفات، الرسائل، النسخ.
- `modules/core/storage.js`: رفع الملفات، التحقق من التخزين، الروابط العامة، استرجاع الملفات القديمة.
- `modules/admin/settings.js`: إعدادات المنصة وأمان حساب المدير عبر Supabase Auth.
- `modules/admin/branding.js`: الشعار والأيقونة والألوان والخطوط والمعاينة والحفظ.
- `modules/admin/backup.js`: إنشاء النسخة والاستعادة الآمنة للكتالوج والإعدادات فقط.
- `modules/store/student-auth.js`: الحساب الاختياري المحلي للطالب مع حفظ تجزئة الرمز وعدم تخزين كلمة المرور كنص.
- `modules/core/config.js`: مصدر واحد لإصدار التطبيق وإعداد Supabase العام.

### تنظيفات إضافية

- إزالة تغليف `adminTab` من التقارير وصحة النظام والنسخ الاحتياطي ومسارات إدارة المندوبين.
- تسجيل الصفحات مباشرة داخل `AlinAdminModules`.
- نقل CSS القديم الذي كان يُحقن من JavaScript إلى `styles/alin-shared.css`.
- تحديث Service Worker وترتيب الملفات ليشمل الخدمات الجديدة.
- النسخ الاحتياطي لا يستبدل الطلبات أو الحسابات أو القيود المالية.
